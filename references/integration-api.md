# Pangolin Integration API Reference

## Authentication

Base path: `/v1`. Authenticated routes use bearer API-key authentication:

```http
Authorization: Bearer <apiKeyId>.<apiKeySecret>
```

Routes list required action permissions. `Root only` means the route also uses `verifyApiKeyIsRoot`; `Limits` means it calls `verifyLimits`.

Successful JSON responses use Pangolin's response envelope unless they stream/export a file:

```ts
{
  data: T | null;
  success: boolean;
  error: boolean;
  message: string;
  status: number;
}
```

## Public Metadata And Health

### `GET /v1/`

Health check.

Auth: none.

Request: no path params, query params, or body.

Response: plain JSON, not Pangolin's envelope:

```ts
{
  message: "Healthy";
}
```

### `GET /v1/docs`

Swagger UI for the generated OpenAPI document.

Auth: none.

Request: no documented path params, query params, or JSON body.

Response: HTML/static Swagger UI responses from `swagger-ui-express`, not Pangolin's envelope.

### `GET /v1/openapi.json`

Returns the generated OpenAPI document as JSON.

Auth: none.

Request: no path params, query params, or body.

Response: raw OpenAPI document JSON generated by `OpenApiGeneratorV3`, not Pangolin's envelope. The generated document uses `openapi: "3.0.0"`, `info.version: "v1"`, `info.title: "Pangolin Integration API"`, and `servers: [{ url: "/v1" }]`.

### `GET /v1/openapi.yaml`

Returns the generated OpenAPI document as YAML.

Auth: none.

Request: no path params, query params, or body.

Response: YAML string generated by `js-yaml` from the OpenAPI document, with response type `application/yaml`; not Pangolin's envelope.
